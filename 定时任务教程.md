# 定时运行爬取任务 — 设置教程

## 方式一：Windows 任务计划程序（推荐）

### 步骤

1. **按 `Win + R`**，输入 `taskschd.msc`，回车打开「任务计划程序」

2. **点击右侧「创建基本任务」**

3. **名称**：`Bilibili收藏夹爬取`，描述可选

4. **触发器**：选择频率
   - 推荐「每周」，选择一天（如周日）
   - 设置时间（如 03:00 凌晨）

5. **操作**：选择「启动程序」
   - **程序/脚本**：点击浏览，选择项目中的 `run_crawler.bat`
   - **起始于**：填写项目目录的完整路径，例如：
     ```
     d:\Justin\0software\crawling-bilibili-personal-collect-main
     ```

6. **完成**：勾选「完成时打开此任务的属性对话框」

7. **在属性中调整**：
   - ✅ 勾选「不管用户是否登录都要运行」（如果需要后台运行）
   - ✅ 勾选「使用最高权限运行」
   - 在「条件」标签中，取消勾选「只有在计算机使用交流电源时才启动」
   - 在「设置」标签中，勾选「如果任务运行时间超过以下时间则停止」→ 设为2小时

### 验证

右键点击创建的任务 →「运行」，确认正常执行。

## 方式二：使用命令行创建计划任务

以管理员身份打开 PowerShell，输入：

```powershell
# 每周日凌晨 3 点运行
schtasks /create /tn "Bilibili收藏夹爬取" /tr "d:\Justin\0software\crawling-bilibili-personal-collect-main\run_crawler.bat" /sc weekly /d SUN /st 03:00 /f
```

### 查看和管理

```powershell
# 查看任务状态
schtasks /query /tn "Bilibili收藏夹爬取"

# 立即运行一次
schtasks /run /tn "Bilibili收藏夹爬取"

# 删除任务
schtasks /delete /tn "Bilibili收藏夹爬取" /f
```

## 注意事项

- Cookie 大约每月过期一次，过期后需要重新运行 `getcookie.py`
- 爬取日志保存在 `bilibili_crawler.log`
- 每次爬取会自动备份旧数据，并清理超过5份的旧备份
